Expected CSV format

Row 1 = month name in column A, Row 2–3 = headers, Row 4+ = data. Only columns A (day) and B (org volume) are read.

March,,,,,, PIN GOAL,Mill 7.5,,DV GOAL,ORG DAILY AVG,EST. ORG DV,UPLIFT Date,Org Volume,Org Increase,Mill 7.5,Need Per Day,Pres,Needed Per Day 1,2022,,111614,3600,151515,4888 2,5778,,107858,3719,145737,5025 3,9422,,104214,3722,142093,5075

Tip: the importer auto-detects the month from cell A1 or the filename. Name your files like march_2026.csv for best results.

Connect Supabase

Until connected, all data lives in your browser's localStorage. Connect Supabase to sync across devices and never lose data.

Setup steps
1Go to supabase.com → New project → name it "OrgTrack"
2Settings → API → copy your Project URL and anon key
3SQL Editor → run the schema below to create the table
4Paste URL + key above → Save & Test → then use Import to load historical data
SQL Schema — run in Supabase SQL Editor
create table org_volume ( id bigserial primary key, entry_date date not null unique, month_num int not null, year_num int not null, day_num int not null, org_volume numeric not null, org_increase numeric, dv_goal numeric default 113636, dv_remaining numeric, org_daily_avg numeric, est_org_dv numeric, uplift_per_day numeric, pin_goal text default 'Mill 7.5', created_at timestamptz default now(), updated_at timestamptz default now() ); alter table org_volume enable row level security; create policy "read" on org_volume for select using (true); create policy "insert" on org_volume for insert with check (true); create policy "update" on org_volume for update using (true); create policy "delete" on org_volume for delete using (true);
REST endpoints used
GET /rest/v1/org_volume?select=*&order=entry_date POST /rest/v1/org_volume ← insert / upsert PATCH /rest/v1/org_volume?id=eq.N ← update DELETE /rest/v1/org_volume?id=eq.N ← delete

All requests use your anon key. The RLS policies above allow public read/write — suitable for personal use. For team use, consider Supabase Auth.
